🍕 Sistema de Gestão para Pizzaria (Engenharia Reversa)

------------------------------------------------------------------------------------------------------------------------------------------------------
📌 Sobre o Projeto

Este projeto tem como objetivo aplicar conceitos de Engenharia Reversa para analisar, compreender e modelar um sistema de gestão de pizzaria.

A proposta consiste em estudar um sistema já existente (ou hipotético), identificar suas funcionalidades, regras de negócio e estrutura, e a partir disso reconstruir sua documentação e possível implementação.

A engenharia reversa é o processo de analisar um sistema pronto para entender seus componentes, funcionamento e lógica interna, permitindo recriar ou melhorar o sistema original  .

------------------------------------------------------------------------------------------------------------------------------------------------------

🎯 Objetivos
Analisar um sistema de pizzaria já existente

Identificar requisitos funcionais e não funcionais

Modelar o sistema (diagramas, fluxos, etc.)

Propor melhorias no sistema

Desenvolver (ou simular) uma solução baseada na análise

------------------------------------------------------------------------------------------------------------------------------------------------------
🧠 Contexto do Problema

Gerenciar uma pizzaria envolve diversos desafios, como:

Controle de pedidos

Gestão de estoque

Organização de entregas

Controle financeiro

Gestão de funcionários

Sem um sistema adequado, esses processos podem gerar erros, atrasos e prejuízos. Sistemas digitais ajudam a automatizar e integrar essas operações, aumentando a eficiência do negócio  .

🧩 Engenharia Reversa Aplicada

No projeto, a engenharia reversa foi utilizada para:

Entender como o sistema funciona

Identificar regras de negócio

Reconstruir documentação inexistente

Criar modelos do sistema

Esse processo permite transformar um sistema “caixa preta” em algo compreensível e reutilizável.
